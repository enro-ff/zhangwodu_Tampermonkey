# 智慧树掌握度自动答题 - 小白使用指南

## 📖 简介

这是一款运行在 **Tampermonkey** 或 **ScriptCat** 浏览器插件上的自动化脚本，旨在帮助用户自动完成智慧树平台课程答题任务。它能自动识别题目、调用 AI 答题、自动点击按钮、自动循环刷题。

> ⚠️ **免责声明**：本工具仅供学习交流使用，请勿用于商业或违规用途。

---

## 🚀 安装步骤

### 第一步：安装浏览器插件

根据你的浏览器选择其一：

| 浏览器 | 推荐插件 | 下载地址 |
|--------|----------|----------|
| Chrome | Tampermonkey | [Chrome 应用商店搜索 Tampermonkey](https://chrome.google.com/webstore) |
| Edge | Tampermonkey | [Edge 应用商店搜索 Tampermonkey](https://microsoftedge.microsoft.com) |
| 360/QQ 等 | ScriptCat | [官网下载](https://docs.scriptcat.org/) |

### 第二步：安装脚本

1. 打开插件管理页面（点击插件图标 → 管理面板）
2. 点击左侧「工具箱」或「获取新脚本」
3. 粘贴本仓库的脚本代码，或直接将脚本文件拖入插件页面
4. 保存后刷新智慧树页面即可生效

### 第三步：确认安装成功

打开智慧树课程页面，如果页面右下角出现一个小面板（显示"掌握度链路"），说明安装成功。

## ⚙️ 首次配置

### 打开设置面板

1. 进入智慧树课程答题页面
2. 点击右下角面板顶部的 **⚙ 按钮**（设置图标）
3. 展开后可以看到三个输入框

### 填写 API 配置（必填）

脚本需要连接 AI 服务来自动答题，你需要提供以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **API Base URL** | AI 服务的接口地址 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| **API Key** | 你的 API 密钥 | `[你的API Key]` |
| **Model Name** | 使用的模型名称 | `qwen-vl-plus` |

> ⚠️ **重要提醒**：由于题目中含有图片，**必须使用具有视觉（Vision）能力的多模态模型**。纯文本模型无法识别图片，可能导致答题失败！
>
> 推荐模型（阿里云百炼）：
> - `qwen-vl-plus` ✅ 有视觉能力（推荐）
> - `qwen3.6-flash-2026-04-16` ⚠️ 无视觉能力（不支持图片）

#### 如何获取这些信息？

以**阿里云百炼（DashScope）**为例：

1. 访问 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 注册/登录账号
3. 进入「模型服务」→ 「API-KEY管理」
4. 创建一个新的 API-KEY，复制保存
5. 在模型市场中选择**带视觉能力的模型**（如 `qwen-vl-plus`），复制模型名称

> 💡 **提示**：不同 AI 服务商的 Base URL 和模型名称格式不同，请参考你使用平台的具体文档。

#### 配置保存后

点击 **「保存配置」** 按钮，配置会保存在浏览器本地（使用 `GM_setValue`），关闭页面后不会丢失，下次打开会自动读取。

## 🎮 界面介绍

### 面板外观

```
┌─────────────────────────┐
│ ● 掌握度链路      ⚙ −  │  ← 标题栏（可拖动）
├─────────────────────────┤
│ API:qwen3.6            │  ← 当前使用的模型（悬停显示完整信息）
├─────────────────────────┤
│ 列表 详情 提升 答题 成绩 │  ← 流程步骤指示器
├─────────────────────────┤
│ [已停止] [循环关] [当前：—] │
├─────────────────────────┤
│  [开始/继续]  [停止]    │
├─────────────────────────┤
│ 12:30:00 待命，点击开始  │  ← 日志区域
└─────────────────────────┘
```

### 各部分说明

- **● 运行指示灯**：运行时绿色闪烁，停止时灰色
- **流程步骤**：显示当前所处环节（列表→详情→提升→答题→成绩）
- **状态标签**：
  - `已停止/运行中`：脚本是否在运行
  - `循环关/循环开`：是否自动循环所有题目
  - `当前：—`：当前识别到的页面类型
  - `API:xxx`：当前使用的 AI 模型
- **日志区域**：显示脚本运行过程中的所有操作和错误信息

## 📋 使用流程

### 基本操作

1. **开始答题**：点击面板的 **「开始/继续」** 按钮
2. **停止脚本**：点击 **「停止」** 按钮
3. **展开/折叠面板**：点击 **「−」** 折叠，点击悬浮按钮 **「ZHS」** 展开
4. **拖动面板**：按住标题栏拖动到任意位置
5. **修改配置**：点击 ⚙ 按钮修改 API 设置

### 典型使用场景

#### 场景一：刷完整门课程

1. 配置好 API（见上方）
2. 确保在掌握度列表页面
3. 点击「开始/继续」
4. 脚本会自动：
   - 找到掌握度低于 80% 的题目
   - 点击进入答题
   - AI 自动答题
   - 自动提交
   - 自动循环下一个题目
   - 直到所有题目都达到 80% 以上
5. 完成后自动停止

#### 场景二：只做单题

1. 手动进入某个题目页面
2. 点击「开始/继续」
3. 脚本只完成当前题目后停止

## 🔧 常见问题与解决

### Q1：脚本没有反应，面板不显示

**可能原因**：
- 脚本未正确安装
- 页面 URL 不在支持列表内

**解决方法**：
1. 确认脚本已保存启用（插件管理面板中显示为绿色）
2. 确认当前页面是智慧树答题相关页面（非列表页、视频页可能不生效）
3. 尝试刷新页面

### Q2：点击开始后显示"AI HTTP 400"错误

**可能原因**：
- API Key 填写错误
- 模型名称不存在
- 账户额度用尽
- Base URL 格式不对

**解决方法**：
1. 点击 ⚙ 检查配置是否正确
2. 打开日志区域，查看详细错误信息（格式如 `AI HTTP 400 | invalid model`）
3. 根据错误提示针对性修改：
   - `invalid model`：模型名称填错了
   - `authentication error`：API Key 错误
   - `quota exceeded`：额度用完了

### Q3：提示"图片长度和宽度不满足模型限制"

**原因**：题目中某张图片分辨率太小（小于 10x10 像素），模型不支持。

**解决方法**：
脚本已内置自动过滤（v1.0.0+），会自动跳过太小的图片。如果仍有问题，可能是网络图片未加载导致尺寸获取失败，可以尝试重新加载页面。

### Q4：AI 答错题/答案不合规

**原因**：AI 答题是概率性的，偶尔会答错或返回格式不对。

**解决方法**：
脚本内置了自动重试机制（最多 3 次），大部分情况下会自动纠错。如果频繁出错，可以尝试：
1. 换一个更可靠的 AI 模型
2. 检查网络连接是否稳定

### Q5：脚本运行到一半卡住了

**可能原因**：
- 页面 DOM 结构变化导致定位失败
- 网络请求超时
- 页面跳转被阻止

**解决方法**：
1. 点击「停止」然后再点「开始/继续」尝试恢复
2. 刷新页面后重新开始
3. 检查日志显示的错误信息

## 📝 高级配置

### 通过油猴菜单配置

除了界面操作，还可以通过插件菜单配置：
1. 点击 Tampermonkey/ScriptCat 图标
2. 选择「设置 API 配置」
3. 依次弹出提示框输入 Base URL、API Key、Model Name

### 清除所有保存的配置

在浏览器控制台（F12）执行：
```javascript
GM_setValue('zhs_api_baseurl', null);
GM_setValue('zhs_api_apikey', null);
GM_setValue('zhs_api_model', null);
location.reload();
```

### 修改循环阈值

脚本默认只刷掌握度低于 80% 的题目。如需修改，找到代码中的：
```javascript
if (!Number.isNaN(pct) && pct < 80) return el;
```
将 `80` 改为你想要的数值。

## 🛡️ 注意事项

1. **账号安全**：API Key 是你的账户凭证，请勿泄露给他人
2. **合理使用**：自动答题可能违反课程平台的使用条款，请酌情使用
3. **网络环境**：部分学校网络可能限制外网访问，导致 AI 请求失败
4. **版本更新**：如遇问题，可检查是否有新版本发布
5. **数据备份**：配置信息保存在浏览器本地，清除缓存会导致配置丢失
